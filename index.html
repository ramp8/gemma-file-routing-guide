Core Architecture

Pipeline zuerst. Fine-tune danach.

Gemma soll nicht PDF-Bytes, Excel-Dateien oder JPG-Pixel direkt „verstehen“. Die robuste Lösung trennt Extraktion, Retrieval, Klassifikation und LLM-Entscheidung.

Develop

Extract

PDF-Text, OCR, Tabellen, Excel-Sheets, JPG-EXIF, Image-Summary und Dateipfad in ein normalisiertes Evidence-Format bringen.

Preview

Retrieve & classify

Ähnliche gelabelte Dateien holen, klassische Baselines trainieren und Top-k Routes mit Confidence erzeugen.

Ship

Gemma decides JSON

Gemma 4 12B lernt: Evidence + Taxonomie + Kandidaten → doc_type, route, confidence, review flag und reason_codes.

Dataset Mix

Mostly synthetic, aber realitätsverankert.

Synthetic Data sollte aus echter Ordnerstruktur, echten Dateinamen und echten Extraktionsartefakten abgeleitet werden. Frei erfundene Beispiele sind nur Ergänzung.

20–40%

Real / weak real

Existierende Folder, Pfade und manuell geprüfte Beispiele als Anchor gegen synthetischen Drift.

30–50%

Synthetic variants

Noise, fehlende OCR, andere Dateinamen, andere Sprachen, Scan statt PDF, reduzierte Evidenz.

10–20%

Hard negatives

Ähnliche Dateien mit anderer Route: Invoice vs Purchase Order vs Supplier Contract.

Modalities

Was pro Dateityp extrahiert wird.

DateitypExtraktionWarum
PDFEmbedded text, Page count, OCR bei Scans, Layout, Tabellen, PDF metadata.Routing hängt oft an Rechnungsnummern, Parteien, Summen, Verträgen und Layoutsignalen.
Excel / CSVSheet names, Header candidates, Column types, Sample rows, Formula count, Pivot/Table-Hints.Große Tabellen werden kompakt repräsentiert; kein riesiger Raw-Dump ins Kontextfenster.
JPG / PNGOCR, Image caption, EXIF, Screenshot/Receipt/Form-Klasse, Vision embedding.Bei textlastigen Bildern ist OCR wichtiger als Vision-Finetuning; VLM-Summary hilft bei visuellen Hinweisen.

Unsloth Docs
Evaluation

Nicht random splitten.

Random Splits überschätzen Performance, weil Vendor, Template oder Folder-Struktur leaken können.

Split by folder

Testet neue Ablagebereiche und verhindert, dass Pfad-Patterns auswendig gelernt werden.

Split by vendor

Rechnungen und Verträge neuer Absender werden realistischer bewertet.

Split by template

Verhindert, dass Layouts statt Dokumenttypen memorisiert werden.

Von Foldern zum Fine-tune.

PhaseDeliverableAkzeptanzkriterium
1. IngestionNormalized records für PDF, Excel, JPG.Jede Datei hat Metadaten, Extraktionsstatus und File-ID.
2. Taxonomiedoc_type, route, sensitivity, workflow_action.Alle Outputs sind per JSON Schema validierbar.
3. Weak labelsFolder-derived Labels plus Audit-Sample.5–10% manuell geprüft, Fehlerklassen bekannt.
4. Synthetic variantsNoise, hard negatives, missing evidence.Judge Score ≥ 4/5, keine Duplikate.
5. BaselinesPath/Text embeddings + classifier.Vergleich gegen Gemma SFT möglich.
6. QLoRA SFTGemma 4 12B evidence-to-JSON model.Top-1/Top-3 Accuracy, Macro F1, Review-Rate gemessen.
